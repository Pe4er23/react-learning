1. Що таке Atomic Design і навіщо ми розділяємо компоненти на атоми та молекули?
Atomic Design — це методологія проєктування інтерфейсів, яка передбачає розбиття UI на базові незалежні частини. Ми розділяємо компоненти на атоми (найдрібніші базові елементи, такі як кнопки чи інпути) та молекули (комбінації атомів, як картки чи поля форм)  для того, щоб створити гнучку структуру, уникнути дублювання та максимально перевикористовувати існуючий код у різних частинах додатку.

2. Як працюють CSS Modules і як вони вирішують проблему глобальних імен класів?
CSS Modules ізолюють стилі на рівні конкретного компонента. Під час збірки проєкту (через Vite) імена класів генеруються унікальними (зазвичай до них додається випадковий хеш). Це гарантує відсутність конфліктів імен класів, навіть якщо в різних компонентах використовуються класи з однаковими назвами (наприклад, .button чи .wrapper).

3. Що таке props.children і в якому компоненті цієї лабораторної роботи ми його використали?
props.children — це спеціальний проп у React, який дозволяє передавати будь-який контент (текст, HTML-елементи або інші компоненти) всередину іншого компонента між його відкриваючим і закриваючим тегами. У цій лабораторній роботі ми використали його у компоненті Card, щоб він міг слугувати контейнером для іншого контенту , а також у компоненті Button для відображення тексту всередині кнопки.

4. Чому атрибут HTML class у JSX записується як className?JSX — це синтаксичне розширення JavaScript. Оскільки слово class є зарезервованим ключовим словом у самому JavaScript (використовується для створення класів в ООП), розробники React прийняли рішення використовувати властивість DOM API className для позначення CSS-класів у розмітці.
