1) Чому використання if/else безпосередньо всередині JSX-розмітки призведе до помилки? 
JSX — це синтаксичний цукор для виклику функцій React.createElement(). Оскільки JSX компілюється у звичайні об'єкти та виклики функцій JavaScript, ти не можеш вставити інструкцію (statement), таку як if/else, всередину виклику функції або об'єкта. Замість цього потрібно використовувати вирази (expressions), які повертають значення (наприклад, тернарний оператор або &&).

2) Яка небезпека криється у використанні числа 0 у лівій частині оператора && в JSX? 
У JavaScript вираз 0 && <Component /> обчислюється і повертає 0 (оскільки 0 є хибним значенням, або "falsy"). React не ігнорує числа під час рендерингу, тому замість того, щоб приховати елемент, він виведе цифру 0 на екран. Щоб цього уникнути, потрібно використовувати строгі булеві перевірки (наприклад, array.length > 0 && ... або !!array.length && ...).

3) Як реалізувати умовний рендеринг, якщо логіка вибору компонента дуже складна (наприклад, 5+ варіантів)? 
У таких випадках краще уникати довгих ланцюжків тернарних операторів чи багатьох &&. Натомість можна використовувати:Інструкцію switch у допоміжній функції перед блоком return.Словник (об'єкт), де ключами є стани, а значеннями — компоненти, наприклад: { list: <StudentList />, stats: <StatisticsData /> }[activeTab].

4) В яких випадках використання тернарного оператора є кращим за використання оператора &&? Тернарний оператор (умова ? А : Б) ідеально підходить, коли у вас є два взаємовиключні стани  (потрібно відрендерити або один компонент, або інший). Оператор && використовується виключно тоді, коли потрібно відрендерити елемент або взагалі нічого.

5) Як React розуміє, що потрібно перерендерити частину інтерфейсу при зміні вкладки? 
Коли ми викликаємо функцію зміни стану (наприклад, setActiveTab), це сигналізує React, що дані компонента змінилися. React запускає процес Reconciliation: він будує новий Virtual DOM, порівнює його з попереднім, знаходить відмінності та точково оновлює лише ту частину реального DOM, яка змінилася (в даному випадку — змінює контент вкладки).
